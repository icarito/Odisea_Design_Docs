# Contribuir a la documentación de Odisea

## Reglas básicas
- Usa enlaces relativos y revisa que no se rompan tras mover archivos.
- Si mueves o renombras archivos, actualiza los enlaces en todo el repo.

## Convenciones
- Usa inglés solo para nombres técnicos o código.
- Prefiere tablas y listas para backlogs y assets.
- Documenta decisiones importantes en AGENTS.md o en el devlog.

## Publicar cambios
- Ejecuta `quarto render` para compilar el sitio.
- Verifica la navegación y enlaces antes de hacer merge.
